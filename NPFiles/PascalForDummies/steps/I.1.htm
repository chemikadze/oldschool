Грамматика языка Паскаль

  Как известно, каждый язык мира имеет свою структуру, грамматику и набор символов. Паскаль, пусть и является языком программирования, не исключение. Начнем с главного: с азбуки программы. В программе можно использовать латинские символы:

ABCDEFGHIJKLMNOPQRSTUVWXYZ
abcdefghijklmnopqrstuvwxyz

Из букв латинского алфавита составляются ИДЕНТИФИКАТОРЫ - слова с определенным смыслом. Надо отметить, что регистр символов не имеет значения. Идентификатор begin можно записать как BEGIN, BegiN, bEGIN.

Так же в тексте программы используются знаки:

_~!@#$()*=-+/<>,.;:'

В каждой программе можно использовать один и тот же набор "слов", который не зависит от реализации и среды, в которой программа написана. Эти слова называют зарезервированными. Их нельзя использовать иначе, кроме как предписывает стандарт языка. Вот они:

  • program
  • begin
  • end
  • if
  • then
  • else
  • case
  • of
  • for
  • to
  • downto
  • do
  • while
  • repeat
  • until
  • string
  • и другие, которые будут освещены позже.

Структура программы

   Каждая программа на языке паскаль состоит из одинаковых частей, располагающихся в строгом порядке. Вот "скелет" программы:

  • имя программы
  • список модулей
  • список новых типов
  • список переменных
  • операторы (сама программа)
Листинг 2.
program ProgramName;
uses 
  Crt;
var
  a:Integer;
begin
  a:=10;
  Write('a=',a);
end.		 {имя программы, может отсутствовать}
{список используемых модулей, может отсутствовать}
{модуль для работы ввода-вывода на экран}
{список переменных, может отсутствовать}
{переменная с именем "a" целого типа}
{начало программы, ОБЯЗАТЕЛЬНАЯ часть}
{оператор присваивания}
{оператор вывода на экран}
{конец программы, ОБЯЗАТЕЛЕН}

   Мы видим программа на Паскале, как и текст на любом другом языке состоит из предложений, но в Паскале они разделяются символом ';'. Обратите внимание на точку после конечного END - она обязательна. Несмотря на достаточно строгий синтаксис, допускаются некоторые "вольности" при написании программы - возможно несколько одинаковых частей (например USES-TYPE-VAR-TYPE-USES-VAR), можно писать текст программы в одну строку, но всё же есть список правил, которые нельзя нарушать.

  • Каждый оператор должен быть определен в коде перед использованием (например, если сначала объявлена переменная типа MyType, а потом сам тип, то это вызовет ошибку компиляции)
  • В качестве имен переменных, типов и др. идентификаторов нельзя использовать зарезервированные слова
  • Нельзя определять несколько идентификаторов с одним именем (например переменная My с типом My)
