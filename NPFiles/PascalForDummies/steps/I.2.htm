Типы данных

   В языке паскаль существует несколько видов типов:

  • целые типы
  • типы чисел с плавающей запятой
  • символьный тип
  • строковый тип
  • массив
  • запись
  • объект

   Рассмотрим каждый вид данных по порядку (3 последние типа мы рассмотрим позже).

Целые типы

   Целые типы - типы данных, значениями которых могут быть только целые числа. Эти типы отличаются друг от друга предельно допустимыми значениями и занимаемой памятью. Записываются они как обычное целое число: 0 125 32 13
Имя типа Диапазон Формат
ShortInt -128..127 со знаком, 8 бит (1 байт)
Integer -32768..32767 со знаком, 16 бит (2 байта)
LongInt -2147483648..2147483647 со знаком, 32 бита (4 байта)
Byte 0..255 без знака, 8 бит (1 байт)
Word 0..65535 без знака, 16 бит (2 байта)
   Целые типы используются для большего числа вычислений (но не всегда). Они занимают мало места, обладают достаточно высокой скоростью при работе с ними. Стоит отметить, что быстрота работы с типами различается. Так, в Turbo Pascal скорость вычисления Integer>Byte>LongInt.

Типы чисел с плавающей запятой

  Числа с плавающей запятой используются достаточно часто, в тех случаях, когда необходимо выполнять точные вычисления с дробными числами. Для их записи существует 2 формата - "обычный" и инженерный. Числа в "обычном" формате записываются так, как ва привыкли их записывать на уроке математики: 10.02 125.512 0.000001 Числа в инженерном формате имеют следующий вид: [+/-]iii.fff[E/e][+/-]eee, где +/-  - указатель знака, iii-целая часть мантиссы, fff - дробная часть мантиссы, eee - количество знаков, на которое перемещена запятая (+ влево, - вправо). Что бы было понятнее, покажем, как нужно переводить числа из одной формы записи в другую.

1258 = 1.258*100 = 1.258*102 = 1.258e2
-1.2e4 = 1.2*104 = 1.2*10000 =12000
0.2e-3 = 0.2*10-3 = 0.2*0.001 = 0.0002

  С записью чисел разобрались, теперь о самих типах.

Тип Диапазон Значащих цифр Размер (байт)
Real +/- 2.9e-39..1.7e38 11-12 6
Single +/- 1.5e-45..3.4e38 7-8 4
Double +/- 5.0e-324..1.7e308 15-16 8
Extended +/- 3.4e-4932..1.1e4932 19-20 10
Comp -9.2e18..9.2e18 19-20 8
  Как видно из таблицы, диапазон чисел с плавающей запятой достаточно велик. Но не стоит обольщаться. Посмотрите на столбец "значащие цифры". Цифры в этом столбце показывают, сколько цифр мантиссы РЕАЛЬНО храниться в памяти. Для примера возьмем тип Real. Он может хранить числа до 17 с 37 нулями. Но если записать туда 30 цифр 2, то сохранятся только 11 первых разрядов, остальное округлиться. Так что нужно быть осторожнее при использовании таких чисел.
  Примечание: тип Comp относится к целым. В некоторых компиляторах он назван как int64 (целый, 64 битный).

Символьный тип данных

  В паскале существует ещё один очень важный тип данных: символьный тип. Как известно, компьютер может хранить в памяти только числа. Но как же тогда хранить символы? Тут поступили просто. Собралась куча профессоров и решили создать единый для всех стандарт хранения символов. Составили таблицу, в которой было указано какой код соответствует какому символу. Всего получилось 256 символов с кодами 0..255. Из них первые 127 одинаковы для всех языков. Это цифра, знаки препинания, буквы английского алфавита и управляющие символы (символы, которые компьютер интерпретирует особенным образом, например 7 это звонок). Для остальных кодов символ зависит от языка.  Символьный тип данных имеет название Char, занимает 1 байт. Символы на языке Паскаль записываются в одинарных кавычках: 'A' '9' '%' так же символ можно записать кодом: #7(звонок) #32(пробел) #65(латинская A). Проводить операции с кодом символа непосредственно нельзя (например запись c:='A'+2 вызовет ошибку).

Строковой тип

   Для записи текстов одних символов недостаточно. Для записи текста используются СТРОКИ. Строки в Паскале - не то же самое, что строка в обычном тексте. Строка в Паскале - последовательность символов, которая может включать в себя символы перевода строки, т.е. текст в несколько строчек. Строка записывается так: 'This is STRING'. Нужно помнить, что длина строки ограничивается 255 символами. Со строками можно проделывать следующие операции: сложение(s1+s2), выделение символа (S[i], где i-номер символа).
